В Cursor Pro можно подключить свою подписку на ChatGPT/​OpenAI, чтобы Cursor использовал именно ваш оплаченный тариф (в т. ч. GPT‑4.1/5, если он доступен в API). Вот ключевые шаги:

Получите API‑ключ OpenAI

Зайдите в консоль OpenAI (https://platform.openai.com/account/api-keys).

Создайте ключ и скопируйте его. Убедитесь, что ваш тариф активен и пополнен.

Настройте Cursor

Откройте Cursor, в левом нижнем углу нажмите на аватар → Settings.

Перейдите в раздел Providers или Connections (в зависимости от версии).

В блоке OpenAI вставьте свой API‑ключ и сохраните.

Выберите модель

В настройках Cursor можно указать модель по умолчанию для чата и автодополнений.

Выберите актуальную модель (если в списке есть GPT‑5 — выберите её; если нет, используйте ближайшую доступную вроде GPT‑4.1).

При работе в чатах проверяйте, что в выпадающем списке модели стоит нужный вариант.

Учтите квоты и лимиты

Cursor будет расходовать ваш баланс OpenAI; следите за использованием в консоли OpenAI.

При превышении лимитов (rate limits, spend cap) запросы начнут ошибаться.

В связке с GitHub

Если хотите, чтобы Cursor работал с репозиториями GitHub под вашей учёткой, подключите GitHub в тех же настройках (Connections → GitHub). Это не обязательно для использования модели, но удобно для работы с проектами.

Проверка

После подключения откройте чат в Cursor, отправьте пробный запрос и убедитесь, что в заголовке/выпадающем списке указана выбранная модель.

В логах (правый верхний угол → View Logs) можно увидеть, вызывает ли Cursor API OpenAI без ошибок.

Если что-то не работает (модель не появляется, ошибки 401/429), проверьте:

Корректность ключа и остаток средств;

Настройки прокси/фаервола;

Вкладку Usage в консоли OpenAI — нет ли превышений.

Этого достаточно, чтобы использовать ваш оплаченный ChatGPT5 внутри Cursor.

Веб-подписка ≠ API-доступ. Подписка «ChatGPT Plus» (или «ChatGPT Team/Enterprise») даёт доступ к моделям в веб-интерфейсе chatgpt.com, но не открывает их автоматически через OpenAI API. Для Cursor нужен именно API-доступ с активным биллингом и разрешёнными моделями.

Новый GPT может быть доступен только вебу. Новые модели (например, GPT‑5) часто сначала появляются во встроенном ChatGPT и не сразу доступны по API. Если в панели OpenAI на вкладке “Models” нужного варианта нет, Cursor его тоже не увидит.

Квоты / Allowlist. Некоторые модели требуют отдельного одобрения или повышенного лимита. Без этого они скрыты. В консоли OpenAI проверьте раздел Usage и Limits; при необходимости подайте запрос на расширение доступа.

Версия Cursor. Убедитесь, что используете свежий релиз Cursor (Help → Check for updates). В старых версиях список моделей может не подтягивать новые названия.

Что можно сделать сейчас:

В консоли OpenAI (https://platform.openai.com) зайдите в Models и посмотрите, какие модели доступны именно через API-ключ. Если там нет GPT‑5, выбирайте ближайшую (GPT‑4.1, o4-mini и т. п.) — Cursor будет работать с ней.

Если вы уверены, что в API-аккаунте есть GPT‑5, но Cursor её не показывает, попробуйте в настройках Providers → OpenAI вручную прописать идентификатор модели (например, gpt-4.1) и перезапустите Cursor.

Проверьте биллинг: на вкладке Usage не должно быть нулевого кредитного лимита или превышения. При необходимости увеличьте spend cap.

Если модель доступна только веб-версии ChatGPT, можно:

ждать, пока OpenAI откроет её через API;

использовать в Cursor «ChatGPT inside Cursor» (левое меню → ChatGPT), но это расходует лимит Cursor Pro, а не вашего API.

Если после проверки остаются вопросы, приложите скрин из OpenAI Console (Models → доступные модели) — по нему будет ясно, доступна ли GPT‑5 через API.

У ChatGPT Team доступ к API не входит автоматически. Чтобы Cursor мог использовать вашу подписку, нужно оформить отдельный API-биллинг в платформе OpenAI. Пошагово:

* Создайте API‑проект. Перейдите на https://platform.openai.com → сверху выберите Projects → New project (при желании можно использовать Default Project).

* Настройте оплату. В новом проекте откройте Billing → добавьте платёжный метод и установите Monthly spend limit (любое значение > 0; иначе вызовы будут блокироваться). Если вы в рабочей организации, убедитесь у администратора, что лимиты разрешены.

* Проверьте доступные модели. Внутри проекта зайдите в Models. Если GPT‑5 там нет, значит по API его пока не выдали — выбирайте модель из списка (GPT‑4.1, o4-mini и т.п.) либо подайте заявку на расширение доступа.

* Сгенерируйте API‑ключ. В API keys → Create new secret key. Скопируйте ключ и держите его в секрете.

* Подключите в Cursor. Settings → Providers → OpenAI → вставьте ключ. Выберите модель из выпадающего списка или пропишите её вручную. Убедитесь, что используете последнюю версию Cursor.

* Протестируйте. Откройте чат в Cursor, отправьте запрос. Если появляется ошибка 401/429 — перепроверьте лимит, ключ и региональные ограничения.

Дополнительно: если аккаунт используется в организации, администратор должен разрешить API-доступ и оплату (иначе раздел Billing будет закрыт). Без официального доступа из консоли OpenAI Cursor не сможет вызывать нужные модели.
